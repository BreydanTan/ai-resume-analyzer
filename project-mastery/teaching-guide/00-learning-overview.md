## 五个学习阶段详解

### 第1阶段：Hello World（1-2天）
**口号**: "让它动起来！"

```
┌─────────────────────────────────┐
│  我要看到第一个成果！            │
├─────────────────────────────────┤
│  1. 装好工具 (Node.js)          │
│  2. 克隆代码 (git clone)        │
│  3. 启动项目 (npm run dev)      │
│  4. 打开浏览器 (localhost:5173) │
│  5. 修改一个字，看到变化!       │
└─────────────────────────────────┘
```

**为什么要这样开始？**
- 快速体验成功感
- 确保开发环境正确
- 看到"代码→浏览器"的自动反应
- 建立信心

**成功标志**: 你能在浏览器看到自己修改的文字。

---

### 第2阶段：理解结构（3-5天）
**口号**: "地图在我脑子里了！"

```
ai-resume-analyzer/
├── app/
│   ├── routes/        ← 所有的"页面"在这里
│   ├── components/    ← 可复用的"模块"
│   ├── lib/           ← 工具和逻辑
│   └── ...
├── project-mastery/   ← 你现在看的文档
└── ...
```

**三个关键理解**:

1. **路由 = 页面**
   ```
   routes/home.tsx    →  首页 (/)
   routes/upload.tsx  →  上传页 (/upload)
   routes/resume.tsx  →  详情页 (/resume/:id)
   ```

2. **组件 = 积木块**
   ```
   <ResumeCard />     →  一个简历卡片
   <Navbar />         →  导航栏
   <ScoreCircle />    →  分数圆圈
   ```

3. **数据 = 云端的笔记本**
   ```
   用户信息  →  云端
   简历数据  →  云端
   评分结果  →  云端
   ```

**成功标志**: 你能指出"上传页面"在哪个文件，"显示评分"的代码在哪行。

---

### 第3阶段：数据库与存储（6-10天）
**口号**: "数据怎么存的，我懂了！"

```
┌──────────────┐
│  你的电脑    │
│  浏览器      │
└────────┬─────┘
         │ HTTPS (安全通道)
         │
┌────────▼──────────────────┐
│  Puter.com 云平台          │
│  ┌───────────────────────┐ │
│  │  KV 存储 (数据库)     │ │
│  │  resume:123 → {...}  │ │
│  │  resume:456 → {...}  │ │
│  └───────────────────────┘ │
│  ┌───────────────────────┐ │
│  │  文件系统 (硬盘)      │ │
│  │  /home/user/a.pdf    │ │
│  │  /home/user/b.pdf    │ │
│  └───────────────────────┘ │
└────────────────────────────┘
```

**关键概念**:

- **KV = Key-Value (钥匙-值)**
  ```
  钥匙: "resume:abc123"
  值: { companyName: "Google", score: 85, ... }
  ```
  就像你的通讯录：名字→电话号码

- **JSON = 结构化数据**
  ```json
  {
    "id": "abc123",
    "companyName": "Google",
    "feedback": {
      "overallScore": 85,
      "tips": [...]
    }
  }
  ```

**成功标志**: 你能用浏览器工具看到云端存储的数据，能解释JSON结构。

---

### 第4阶段：核心功能（2-3周）
**口号**: "功能是怎么实现的，我会改了！"

```
用户认证  →  简历上传  →  AI分析  →  显示结果
   ↓           ↓          ↓         ↓
 Puter      Puter.js    Claude   网页显示
```

**5个关键流程**:

1. **认证**: 用户点击登录 → Puter弹出登录框 → 用户输入 → 返回用户信息
2. **上传**: 用户选择PDF → 上传到云端 → 云端生成缩略图 → 保存元数据
3. **分析**: 简历发给Claude AI → AI阅读并评分 → 返回JSON格式的评分
4. **存储**: 评分结果保存到KV存储 → 用户下次打开能看到历史
5. **显示**: 从云端取数据 → 在网页上美化展示 → 用户看到漂亮的报告

**成功标志**: 你能解释"简历评分"的完整流程，能用AI协助修改AI提示词。

---

### 第5阶段：功能扩展（4周+）
**口号**: "我要添加自己的功能！"

```
现有功能 (上传→评分)
  ↓
想象新功能 (导出PDF、分享、对比)
  ↓
参考现有代码
  ↓
用AI协助写代码
  ↓
测试新功能
  ↓
🎉 完成!
```

**能做的3个实战项目**:

1. **导出报告** - 把评分结果导出为PDF
2. **简历对比** - 上传两份简历，对比分数
3. **改进建议** - 根据AI反馈生成一个改进清单

**成功标志**: 你成功添加了一个全新功能，并且能向别人解释它是怎么工作的。

## 学习的3个秘诀

### 秘诀1️⃣: 边学边做
```
❌ 不要: 先把所有理论学完，再开始做
✅ 要: 做→遇到问题→查资料→理解→继续做
```

### 秘诀2️⃣: 用AI协助，不是替代
```
❌ 不要: 让AI帮你写所有代码
✅ 要: 自己理解流程，让AI帮助你编写具体代码
```

示例好的方式:
- "我想在这里加一个按钮，怎样修改代码？"
- "这个错误什么意思？"
- "能解释这段代码吗？"

### 秘诀3️⃣: 定期回顾
```
做完一个任务后:
  1. 关闭所有代码
  2. 从头解释你做了什么
  3. 用自己的话写下来（或录音）
  4. 第二天再看一遍
```

## 常见困境与解决方案

### 困境1: "代码太多了，我从哪里开始？"
**解决**: 按照学习阶段顺序，不要跳过。每个阶段都建立在上一个基础上。

### 困境2: "我看不懂这些术语..."
**解决**: 查阅概念词典（02-concept-dictionary.md）。所有术语都有日常类比。

### 困境3: "我写的代码不工作！"
**解决**: 按照调试决策树（resources/debugging-flowchart.md）一步步排查。

### 困境4: "我觉得进度太慢..."
**解决**: 这很正常！编程就是这样的。每个人在第2周都觉得慢。第3周开始就会加速。

### 困境5: "我是不是学不会编程？"
**解决**: **你能的！** 这个项目设计给完全没有基础的人。坚持到第3周，你会惊讶自己学了多少。
