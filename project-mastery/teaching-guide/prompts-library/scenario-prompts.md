# 场景化提示词 | Scenario Prompts

## 🎯 10个常见场景的完整提示词

每个场景都是一个真实的开发任务。复制相应的提示词并修改具体细节，就能快速得到AI的帮助。

---

## 场景1️⃣: 添加一个新页面

**真实场景**: 你想在应用中添加一个"设置"页面。

### 完整提示词

```
我想在 AI 简历分析项目中添加一个新页面。

新页面信息:
- 页面名: 设置 (Settings)
- 路由: /settings
- 功能: 显示用户设置，如语言、主题等
- 何时显示: 用户点击导航菜单中的"设置"

现有结构:
- 其他页面在: app/routes/
- 路由配置在: app/routes.ts
- 导航栏在: app/components/Navbar.tsx

能给我:
1. 新的 settings.tsx 文件的基础代码
2. routes.ts 中需要添加的配置
3. Navbar.tsx 中需要添加的菜单项
4. 可能需要的 store 修改（如果涉及全局状态）
5. TypeScript 类型定义

同时:
- 确保与现有代码风格一致
- 包括权限检查（未登录时不能访问）
- 包括基本的 Tailwind 样式
```

## 场景2️⃣: 修改组件样式

**真实场景**: ResumeCard 组件太丑了，你想改进它的设计。

### 完整提示词

```
我想改进 ResumeCard 组件的样式。

当前文件: app/components/ResumeCard.tsx

当前问题:
1. [描述当前的样子]
2. [用户反馈]
3. [你的想法]

想要达到:
- 更现代化的外观
- 更好的视觉层次
- 更强的交互反馈

具体改进:
1. [改进1：比如添加hover效果]
2. [改进2：比如改进排版]
3. [改进3：比如改进颜色搭配]

能给我:
1. 改进后的 className（使用Tailwind CSS）
2. 任何需要的额外的 CSS
3. 交互效果（hover、click等）
4. 不同状态的样式（loading、error等）

注意:
- 保持组件的功能不变
- 确保响应式设计
- 保持与项目整体风格一致
```

## 场景3️⃣: 添加表单验证

**真实场景**: 上传页面的表单需要验证用户输入。

### 完整提示词

```
我想给上传表单添加验证。

表单字段:
1. jobDescription (职位描述) - 必填，最少10字，最多500字
2. companyName (公司名) - 必填，长度1-50
3. jobTitle (职位名) - 必填，长度1-50
4. resumeFile (简历文件) - 必填，仅接受PDF

验证规则:
[列出所有规则]

想要的用户体验:
- 实时验证（边输入边验证）
- 显示清晰的错误信息
- 禁用提交按钮直到表单有效
- 成功提交后清空表单

能给我:
1. 验证逻辑的代码
2. 错误信息的显示组件
3. 表单状态管理的代码
4. 提交处理的代码
5. 测试用例

使用:
- 不使用第三方表单库
- 直接用 React hooks
- 与现有代码风格一致
```

相关代码:
[粘贴上传和保存的代码]

已尝试的诊断:
1. [尝试过的诊断和结果]
2. [尝试过的诊断和结果]

我的理论:

能帮我:
1. 诊断这个问题
2. 找出根本原因
3. 提供修复方案
4. 提供防止再发生的方案

诊断步骤:
- 在哪里添加日志？

## 场景6️⃣: 优化性能

**真实场景**: 有很多简历时，页面加载很慢。

### 完整提示词

```
我的应用性能不好。

性能指标:
- 首次加载: [多少秒]
- 加载100份简历: [多少秒]
- 切换页面: [多少秒]

瓶颈识别:
[使用DevTools找到的慢操作]

代码:
```typescript
[影响性能的代码片段]
```

想要达到的指标:
- 首次加载: < 2秒
- 列表切换: < 100ms
- 详情页加载: < 1秒

能给我优化建议吗？

考虑的优化方向:
1. 数据加载优化（分页、按需加载）
2. 组件优化（memo、懒加载）
3. 缓存策略
4. 虚拟列表（如果有长列表）
5. 资源压缩

还要提供:
- 每个优化的预期效果
- 实现复杂度
- 可能的副作用
```

## 场景7️⃣: 与AI协作添加功能

**真实场景**: 你想要一个"一键对比"功能但不知道从哪开始。

### 完整提示词

```
我想添加一个新功能到这个项目。

功能需求:
- 名称: 一键对比
- 描述: 用户选择两份简历，并排对比评分
- 访问: 在简历详情页添加一个"对比"按钮

我的技能水平:
- 能创建基础组件
- 能理解数据流
- 但不确定整体架构

能给我一个完整的实现计划吗？

包括:
1. 需要创建/修改哪些文件
2. 数据怎么流动
3. 每一步的具体代码
4. 怎样集成到现有系统
5. 测试方案

同时给出:
- 预计实现时间
- 难度等级
- 常见陷阱和解决方案
```

## 场景8️⃣: 设计数据模型

**真实场景**: 你想添加"用户收藏"功能，需要设计数据结构。

### 完整提示词

```
我想添加"收藏简历"功能。

需求:
- 用户可以标记简历为"已收藏"
- 收藏状态需要保存到云端
- 首页显示"只看已收藏"的过滤选项
- 在简历卡片显示收藏图标

能帮我设计数据模型吗？

需要考虑的:
1. 怎样存储收藏状态？
2. 数据结构怎样设计？
3. KV键名怎样命名？
4. 怎样保持与Resume的关联？
5. 怎样处理删除Resume时的收藏？

能给我:
1. TypeScript 接口定义
2. KV 存储的键名规范
3. CRUD 操作的实现
4. 前端集成的代码
5. 可能的问题和解决方案
```

## 场景9️⃣: 测试和质量保证

**真实场景**: 你添加了新功能，想确保它工作正常。

### 完整提示词

```
我完成了一个新功能的实现。

功能: [功能名称]

代码:
```typescript
[新功能的主要代码]
```

现在我想测试它。

能给我一个完整的测试计划吗？

包括:
1. 应该测试哪些场景？

3. 边界情况: [列出]
4. 错误情况: [列出]
5. 用户交互测试: [列出]

对于每个场景:
- 具体的测试步骤
- 预期结果
- 怎样验证

同时:
- 怎样在浏览器手动测试

## 场景🔟: 部署和发布

**真实场景**: 你要把应用部署到网上让别人用。

### 完整提示词

```
我完成了项目的开发，想要部署到网上。

当前状态:
- 在本地 npm run dev 工作正常
- 代码在 GitHub
- 想让别人可以访问

部署需求:
- 需要注册域名吗？
- 需要付费吗？
- 能免费部署吗？

能给我一个完整的部署方案吗？

包括:
1. 推荐的部署平台
   - 费用如何？
   - 优缺点是什么？
2. 具体的部署步骤
3. 部署前需要做什么准备？
4. 部署后怎样监控？
5. 出问题怎样回滚？

同时考虑:
- 环境变量管理
- 数据库/存储配置
- SSL/HTTPS
- 日志和监控
- 备份策略

推荐方案:
- 最简单的方案（快速上线）
- 最便宜的方案
- 最专业的方案
```

## 💡 提示词定制技巧

### 如果你的情况更复杂

添加这一部分：
```
额外的背景信息:
[描述特殊的限制或要求]

我的约束:
- 只能用这些库: [列表]
- 不能修改的地方: [列表]
- 其他限制: [列表]
```

### 如果你想要更详细的回答

添加这一部分：
```
我想要非常详细的回答。

包括:
- 每个文件中的完整代码
- 详细的解释为什么这样写
- 可能的替代方案和权衡
- 最佳实践和陷阱
```

### 如果AI的回答太长或太短

告诉AI：
```
前面的回答太长/太短了。

能给我:
- 核心要点总结 (3-5点)
- 或更详细的步骤 (10个以上)
```
