# 调试提示词 | Debugging Prompts

## 🐛 当代码出问题时

这些提示词帮你快速诊断和修复问题。

---

## 错误信息理解

### 提示词1: 我看不懂错误信息

```
我在浏览器console看到这个错误：

[粘贴完整的错误信息，包括Stack trace]

能用人类能理解的语言解释这个错误吗？

特别是:
1. 这个错误什么意思？
2. 通常是什么原因引起的？
3. 我应该看哪个文件？
4. 怎样修复？

我的上下文:
- 我在做什么操作时看到的
- 最近改了什么代码
- 其他可能的信息
```

### 提示词2: 错误信息很长

```
我的代码出现了一个错误，但错误信息很长。

能帮我提取关键信息吗？

完整的错误输出:
[粘贴整个错误]

问题是什么？
```

## 综合诊断

### 提示词17: 完整的诊断

```
我的项目出现了一个复杂的问题，想系统地诊断。

症状:
[详细描述问题现象]

何时出现:
[什么时候、什么条件下]

影响范围:
[影响了什么功能]

错误信息:
[如果有的话]

已尝试:
1. [尝试过的]
2. [尝试过的]
3. [尝试过的]

环境:
- Node: [版本]
- npm: [版本]
- 浏览器: [浏览器]
- OS: [操作系统]

代码:
[相关的代码片段，大概50-100行]

能帮我系统地诊断吗？

诊断步骤:
1. [第一步]
2. [第二步]
3. [第三步]
```

### 提示词18: 一步一步调试

```
我想一步一步地调试这个问题。

问题:
[简短描述]

首先，我想理解：
[你最想理解的部分]

能引导我一步一步地诊断吗？

包括:
1. 第一步应该检查什么？
2. 如何检查？
3. 如果是这样，下一步是什么？
4. 如果不是这样，下一步是什么？
```

## 性能和优化

### 提示词19: 性能分析

```
我想优化项目的性能。

现在的情况:
- 某个功能很慢: [功能名]
- 需要: [多少秒]
- 理想: [多少秒]

代码:
```typescript
[相关的函数]
```

能帮我分析性能瓶颈吗？

检查清单:
1. 是否有不必要的重新渲染？
2. 是否有大量的数据处理？
3. 是否有网络请求优化的空间？
4. 是否可以使用缓存？
```

```
我怀疑有内存泄漏。

- 浏览器的内存占用越来越大

代码:
```typescript
[可能导致泄漏的代码]
```

怎样找出和修复内存泄漏？
```
