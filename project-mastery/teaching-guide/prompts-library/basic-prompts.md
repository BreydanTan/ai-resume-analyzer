# 基础操作提示词 | Basic Prompts Library

## 📚 如何使用本库

这里收集了**可以直接复制使用**的AI提示词。每个提示词都设计为：
- 清晰有效
- 容易定制
- 快速得到答案

**使用方法**:
1. 找到你的场景
2. 复制整个提示词
3. 如果需要，修改中括号内的内容
4. 粘贴到Claude、ChatGPT等AI工具

---

## 🛠️ 环境搭建

### 提示词1: Node.js安装困难

**场景**: 无法安装Node.js

**提示词**:
```
我正在尝试安装Node.js用于[项目名],但遇到了问题。

我的操作系统: [Windows/Mac/Linux]

我遇到的错误: [粘贴完整错误信息]

我已经尝试过:
1. [尝试过的方法]
2. [尝试过的方法]

能给我一个分步的解决方案吗？
```

### 提示词2: npm install失败

**场景**: 安装依赖时出错

**提示词**:
```
运行 `npm install` 时出现了这个错误：

[粘贴完整的错误输出]

这是什么意思？怎么修复？

我的Node版本: [输出 node --version 的结果]
我的npm版本: [输出 npm --version 的结果]
```

### 提示词3: 项目无法启动

**场景**: npm run dev 或 npm start 不工作

**提示词**:
```
当我运行 `npm run dev` 时，看到这些错误：

[粘贴错误信息]

能帮我诊断并修复这个问题吗？

我的环境:
- Node版本: [版本号]
- 操作系统: [系统]
- 项目路径: [路径]

我的排查步骤:
1. [已经尝试的]
2. [已经尝试的]
```

## 📄 文件操作

### 提示词4: 找不到某个文件

**场景**: 不知道文件在哪里

**提示词**:
```
在AI简历分析项目中，我找不到 [文件名或功能] 的代码。

我已经搜索过:
1. app/ 文件夹
2. lib/ 文件夹
3. components/ 文件夹

能告诉我 [文件名或功能] 应该在哪个文件里吗？

项目结构:
app/
├── routes/
├── components/
├── lib/
└── ...
```

### 提示词5: 理解文件的作用

**场景**: 不懂某个文件做什么的

**提示词**:
```
我在看 [文件路径] 这个文件，但不太懂它做什么。

[粘贴文件的开头部分，大概20-30行]

能用简单的语言解释这个文件的作用吗？

特别是:
1. 这个文件导出了什么？
2. 其他文件怎么使用它？
3. 它依赖了什么？
```

### 提示词6: 修改文件

**场景**: 想改某个文件但不知道怎么改

**提示词**:
```
我在 [文件路径] 中想做一个修改。

目标: [描述你想做什么改动]

当前代码:
```typescript
[粘贴相关的代码块，大概10-20行]
```

应该怎么修改？

注意:
1. 不要破坏其他功能
2. 要保持代码风格一致
3. [其他特殊要求]
```

## 💡 快速参考

### 直接问AI的好问题格式

```
问题: [简短描述]

背景: [上下文]

代码: [相关代码片段，如果有]

我的理解: [你已经了解的]

我想知道: [具体问题]

我已经尝试: [尝试过的方法]
```

### 容易被回答的提示词特征

✅ **好的特征**:
- 明确的目标
- 具体的代码片段
- 清晰的错误信息
- 你已经尝试过的方法

❌ **不好的特征**:
- 太笼统 ("代码不工作")
- 没有具体信息
- 没有上下文
- 一张大截图（复制成文字更好）
